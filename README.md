### Introduction

This repository contains the source and supporting documentation for my MTSS Decision-Support System case study.

The project explores how user research, workflow design, and systems thinking can improve complex educational processes by reducing cognitive load while preserving professional judgment.

Rather than replacing educator expertise, the system provides structure, documentation, and guided decision support so users can focus on the decisions that require human judgment.

---

### The Problem

Schools often rely on lengthy documentation, scattered resources, and institutional knowledge to guide MTSS implementation.

While experienced educators eventually learn the process, new staff frequently struggle with:

- knowing where to begin
- locating required documentation
- understanding district procedures
- identifying appropriate next steps
- maintaining consistency across teams

The result is unnecessary cognitive load, inconsistent implementation, and slower support for students.

---

### What I Built

I designed a workflow-centered decision-support system informed by educator interviews and survey data.

Rather than presenting another handbook, the project organizes MTSS into guided workflows that help educators navigate the process while preserving local flexibility.

The work emphasizes:

- workflow design
- information architecture
- user-centered research
- implementation planning
- documentation strategy
- human-centered AI principles

---

### Design Philosophy

Several principles guided the design:

- Research before building.
- Structure reduces cognitive load.
- The system carries the process; educators make the decisions.
- Documentation is part of the product—not an afterthought.
- Good workflows improve consistency without removing professional judgment.
